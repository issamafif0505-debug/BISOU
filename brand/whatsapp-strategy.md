### C. Automation Tools Recommandés

**Option 1 : Botpress (Gratuit/Premium)**
- Avantage : Interface simple, chatbot IA
- Coût : Gratuit jusqu'à 100 conversations/jour
- Setup : 2-3 heures
- Lien : botpress.com

**Option 2 : Typebot (Recommandé pour ÔRIYA)**
- Avantage : Visual workflow builder, pas de code
- Coût : Gratuit jusqu'à 500 contacts
- Setup : 1-2 heures
- Lien : typebot.io
- Best for : Catalogues produits + FAQ

**Option 3 : Make.com (Zapier alternative)**
- Avantage : Intégration CRM + Google Sheets + payment
- Coût : 9$/mois starter
- Setup : 3-4 heures
- Lien : make.com
- Best for : Workflow avancés + inventory management

**Option 4 : OpenChat (Solution WhatsApp native)**
- Avantage : Directement dans WhatsApp Business
- Coût : Gratuit
- Limitation : Moins customizable

**RECOMMANDATION ÔRIYA** : Combiner Typebot (chatbot) + Make.com (workflows)

## 4. AUTOMATION WORKFLOW (Visual Flows)

### Flow 1 : First Contact / Welcome

```
TRIGGER : Customer sends ANY message
   ↓
STEP 1: Send Welcome Message
   └→ "🌟 Bienvenue chez ÔRIYA..."
   └→ Display Catalog Link
   └→ Add to CRM as "New Lead"
   ↓
STEP 2: Immediate Chatbot Question
   └→ Bot asks: "Quel pack t'intéresse?"
   └→ Options: Pack Découverte / Cadeau / Mariée / Mini / Autre
   ↓
STEP 3: Auto-set Reminder
   └→ IF no response in 1 hour → Send follow-up
   └→ Message: "Besoin d'aide pour choisir?"
   ↓
STEP 4: Offer Personal Assistance
   └→ "Je peux te faire des recommandations!"
   └→ Ask: "Budget? Style? Occasion?"
   ↓
END FLOW

AUTOMATION TOOL: Typebot (Free tier)
```

### Flow 2 : Product Interest → Cart Abandonment Sequence

```
TRIGGER: Customer asks about product OR clicks product in catalog
   ↓
STEP 1: Send Product Details Message
   └→ Price, description, specs, certification
   └→ Delivery info
   └→ Guarantee info
   ↓
STEP 2: Check Interest Level
   └→ Bot asks: "Intéressé?"
   └→ Options: OUI / NON / VOIR AUTRES
   ↓
IF "OUI" :
   STEP 3A: Request Confirmation
   └→ "Super! Confirme ta commande?"
   └→ "Quelle est ton adresse?"
   └→ Collect: Name, Address, Phone
   ├→ GO TO: Flow 3 (Purchase)

IF "NON" or "VOIR AUTRES":
   STEP 3B: Suggest Alternatives
   └→ "Regarde ce pack, tu pourrais aimer..."
   └→ Show similar products
   ↓
STEP 4: Set Auto-Reminders (IF cart abandoned)
   └→ +6h : "Toujours intéressé?"
   └→ +24h : "Dernière chance, stock limité!"
   └→ +48h : "On libère le stock pour d'autres clients"
   ↓
END FLOW / OR GO TO: Flow 3

AUTOMATION TOOL: Make.com (Conditional Logic)
```

### Flow 3 : Purchase → Order Confirmation → Delivery

```
TRIGGER: Customer says "OUI" or confirms purchase
   ↓
STEP 1: Confirmation Message
   └→ Send: "✅ COMMANDE CONFIRMÉE"
   └→ Include: Order ID, Price, Delivery Details
   └→ Add to CRM as "Customer - Active"
   ↓
STEP 2: Payment Confirmation
   └→ WhatsApp message: "Paiement à la livraison?"
   └→ Confirm payment method
   └→ Add to invoice tracking
   ↓
STEP 3: Pre-Delivery (4-6h before estimated delivery)
   └→ Send: "Le livreur vous appellera dans 4h"
   └→ Include: Driver phone (if available)
   └→ Offer: Alternative delivery time if needed
   ↓
STEP 4: Post-Delivery Immediate (+2h after delivery)
   └→ Send: "Tu as reçu? Comment ça va?"
   └→ Option to rate experience
   ↓
STEP 5: Survey & Review Request (+24h after delivery)
   └→ Send: "Comment tu trouves ÔRIYA?"
   └→ 5-star rating request
   └→ Photo challenge (10% off next order)
   ↓
STEP 6: Repeat Purchase Trigger (+7 days)
   └→ IF "repeat customer ready" in CRM
   └→ Send special offer or loyalty message
   ↓
END FLOW

AUTOMATION TOOL: Make.com + Zapier (Complex workflow)
```

### Flow 4 : Payment Issue Resolution

```
TRIGGER: Customer says "problème paiement" OR delivery failed payment
   ↓
STEP 1: Escalate to Manager
   └→ Flag in CRM as "URGENT - Payment Issue"
   └→ Send to manager's dashboard
   └→ Set SLA: Respond < 1 hour
   ↓
STEP 2: Offer Alternative Payment Methods
   └→ Option 1: Wafacash payment link
   └→ Option 2: Bank transfer (if available)
   └→ Option 3: Reschedule delivery
   └→ Option 4: Split payment
   ↓
STEP 3: Confirm New Payment Method
   └→ Execute payment
   └→ Send receipt/confirmation
   └→ Update delivery schedule if needed
   ↓
STEP 4: Compensation
   └→ IF delivery rescheduled: Send 5-10% voucher
   └→ Message: "Désolés pour le problème, voici 5% off"
   ↓
END FLOW

AUTOMATION TOOL: Make.com (High priority)
```

### Flow 5 : Negative Feedback / Service Recovery

```
TRIGGER: Customer expresses dissatisfaction / bad product / late delivery
   ↓
STEP 1: Immediate Acknowledgment
   └→ MANUAL response within 30 minutes
   └→ Show empathy: "Désolés! C'est pas normal"
   └→ Message: "😔 On prend ça au sérieux!"
   ↓
STEP 2: Root Cause Analysis
   └→ Ask: "Quel est le problème exactement?"
   └→ Document everything
   └→ Escalate to manager
   ↓
STEP 3: Offer Solution
   └→ IF product defect → Full refund or replacement
   └→ IF late delivery → 10% voucher + apology
   └→ IF packaging issue → Compensation
   └→ IF damaged item → Immediate replacement
   ↓
STEP 4: Get Agreement
   └→ "Ça te convient?"
   └→ Wait for customer confirmation
   ↓
STEP 5: Follow-up (after resolution)
   └→ +48h : "Tout va mieux?"
   └→ "On a changé ça pour ne plus que ça arrive"
   └→ Offer special gesture (future discount)
   ↓
STEP 6: Recover Customer
   └→ IF resolved well → Convert back to loyal customer
   └→ Send: "Merci de nous donner une 2nde chance"
   └→ Exclusive offer for comeback
   ↓
END FLOW

AUTOMATION TOOL: Make.com (Manual + Auto)
```

## 5. TAUX & METRICS À TRACKER

### A. Daily/Weekly KPIs

```
📊 DAILY DASHBOARD (à vérifier chaque matin)

Messages Reçus/Jour
   Cible : 50+ messages/jour
   Minimum viable : 30+ messages/jour
   Mesure : Total dans Insights WhatsApp

Response Time Moyen
   Cible : < 2 heures
   Minimum acceptable : < 4 heures
   Mesure : Timestamp 1st message vs 1st reply

Messages Unanswered
   Cible : 0 unanswered messages
   Minimum : < 2% des messages
   Mesure : Total messages - Total réponses

First Response Rate
   Cible : > 85%
   Acceptable : > 80%
   Mesure : Messages répondus / Messages reçus

Customer Satisfaction Score
   Cible : > 90% (from surveys)
   Minimum : > 85%
   Mesure : Post-purchase survey rating
```

### B. Weekly/Monthly Business KPIs

```
📈 CONVERSION METRICS (Semaine/Mois)

Conversion Rate WhatsApp
   Cible : 15-20% (Inquiries → Purchases)
   Minimum : 10%
   Calcul : (Purchases from WhatsApp / Total Inquiries) × 100

   Example:
   100 product inquiries → 15 purchases = 15% CR
   ✓ Excellent if > 18%
   ⚠️  Good if 15%
   ❌ Needs improvement if < 12%

Average Order Value (AOV)
   Cible : 250 MAD
   Current expected : 229-449 MAD (packs range)
   Calcul : Total Revenue / Total Orders

   Strategy to increase AOV:
   - Suggest "Pack Up" (upgrade)
   - Bundle offers
   - Free shipping threshold

Repeat Purchase Rate
   Cible : 25% (2nd purchase within 90 days)
   Minimum : 15%
   Calcul : Customers with 2+ orders / Total customers

   Strategy to increase:
   - Post-purchase loyalty messaging
   - Referral program (50 MAD credit)
   - Birthday/Anniversary offers

Customer Lifetime Value (CLV)
   Cible : 500+ MAD (2-3 purchases average)
   Calcul : Average customer spends X MAD over Y months

   Target segment analysis:
   - New customers : 229 MAD (1st purchase)
   - Repeat customers : 250+ MAD (2nd)
   - Loyal customers : 500+ MAD (3+)

Churn Rate
   Cible : < 15% (customers who don't return)
   Mesure : Customers not returning in 90 days

   Intervention triggers:
   - 60 days no purchase → Send "Ça te manque?"
   - 90 days no purchase → "On a des nouveaux!"
   - 120 days → Exit recovery attempt
```

### C. Engagement Metrics

```
💬 ENGAGEMENT TRACKING

Messages Per Conversation
   Cible : 8-12 messages per customer journey
   Mesure : Total messages / Total conversations

   Breakdown:
   - Minimal: 2-3 messages (quick sale)
   - Engaged: 5-8 messages (questions asked)
   - High-touch: 10+ messages (needs support)

Response Rate to Automation
   Cible : > 60% respond to first auto-message
   Mesure : % customers who reply to welcome

### C. Message Sending Best Practices

```
✍️  MESSAGE SENDING BEST PRACTICES

TIMING:
├─ Peak hours: 18h-20h (highest engagement)
├─ Second peak: 12h-13h (lunch break)
├─ Avoid: 7h-9h (morning rush), 23h+ (sleep)
├─ Best day: Saturday > Friday > Sunday
└─ Worst day: Monday 7-11h

FREQUENCY:
├─ Max per day: 2-3 personal messages
├─ If more: Use automation (chatbot)
├─ Don't spam same customer > 1x/day
├─ Abandoned cart: Follow-up at 6h + 24h only
└─ Promo: 1x per week MAXIMUM

TONE:
├─ Friendly, casual (not corporate)
├─ Use names when possible ("Hey Fatima!")
├─ Emojis OK (but not too many)
├─ Short paragraphs (3-4 lines max)
├─ Conversational (like texting friend)
└─ Authentic (not salesy)

EMOJI RULES:
├─ Use 1-3 per message (not 10+)
├─ Relevant to message content
├─ Brand consistent (gold, heart, ✨)
├─ Example: "💛" for love/brand, "📦" for shipping
├─ Avoid: 🔥 (too salesy), 😭 (negative)
└─ Test: What resonates with YOUR audience

COPY STRUCTURE:
├─ Hook (first line grabs attention)
├─ Info (what they need to know)
├─ CTA (call-to-action: OUI / DEMANDE / COMMANDE)
├─ Warmth (end with warmth, not urgency)
└─ Always: Leave question mark open (drives response)

MOBILE OPTIMIZATION:
├─ Mobile first (all users on phone!)
├─ Short lines (no long paragraphs)
├─ Break into bullets
├─ Spacing matters (visual breathing room)
├─ Preview on small screen BEFORE sending
└─ Test send to yourself first
```

├─ Winner: Usually shorter wins (mobile)

├─ Version A: "Intéressé? Tapez OUI 👍"

├─ Metric: Response rate
├─ Winner: More conversational usually wins

Test Timeline:
├─ Test period: 1 week minimum
├─ Sample size: 20+ conversations
├─ Track winner for 2 weeks
├─ Then test new element
├─ Keep winning elements (compound winners)

## 10. TROUBLESHOOTING & FAQ FOR MANAGERS

### A. Common Operational Issues

```
❌ PROBLEM: Too many messages, can't keep up

SOLUTIONS:
1. Implement Typebot chatbot (deflects 50% of inquiries)
2. Use auto-reply templates for common Q's
3. Batch response time: Respond every hour, not immediate
4. Hire assistant (split shifts)
5. Limit inquiry windows: "Available 7h-22h only"
6. Aggressive abandoned cart automation (reduces follow-ups)

═══════════════════════════════════════════════════════

❌ PROBLEM: Low conversion rate (<10%)

ROOT CAUSES:
├─ Product not clear (add better description)
├─ Price too high (test lower promo)
├─ Trust issue (add customer reviews/testimonials)
├─ Process confusing (simplify steps)
├─ Payment friction (easy payment!)
└─ Bad timing (follow up when customer online)

FIXES (in order):
1. Add 3-5 customer testimonials to first message
2. Lower starter pack to 199 MAD (psychological pricing)
3. Add "Money-back guarantee 7 days free" to product
4. Simplify: Just ask for address, no extra steps
5. Emphasize easy payment: "Paiement à la livraison"

═══════════════════════════════════════════════════════

❌ PROBLEM: High cart abandonment (>75%)

ROOT CAUSES:
├─ Customer wanted more info (didn't provide)
├─ Unexpected shipping cost (mention upfront)
├─ Forgot about cart (no reminder)
├─ Payment anxiety (didn't feel secure)
└─ Price regret (asked but no confidence)

FIXES:
1. Add full product details in first message (not secondary)
2. Mention shipping cost in product message (no surprises)
3. Auto-reminder at 6h + 24h (mandatory, aggressive)
4. Add "Guarantee 7 days" + "No questions asked" to product
5. Chat follow-up: "Still interested? Happy to help!"

═══════════════════════════════════════════════════════

❌ PROBLEM: Customer complaints about delivery time

ROOT CAUSES:
├─ Wrong expectation set (said 24h, took 48h)
├─ Livreur didn't call (customer not ready)
├─ Address confusion (wrong building)
└─ Time window too wide (customer missed)

FIXES:
1. Always say "24-48h" not "24h" (underpromise)
2. Call 4h before delivery (not 30 min before)
3. Confirm exact address & building on 3rd time
4. Give 2-hour window "Will call between 16-18h"
5. Message day-of: "Livreur arrive aujourd'hui 4-6pm"

═══════════════════════════════════════════════════════

❌ PROBLEM: Product quality complaints

ROOT CAUSES:
├─ Manufacturing defect (rare, <2%)
├─ Care instructions not followed
├─ Unrealistic expectations (not real gold)
└─ Damage during delivery (packaging)

FIXES:
1. Add care instructions in delivery package
2. Add disclaimer: "Plaqué, not solid gold" clear in product
3. Better packaging (more bubble wrap, padding)
4. Quality control: Inspect 10% of orders before shipping
5. Auto-reply with care tips in post-delivery message

═══════════════════════════════════════════════════════

❌ PROBLEM: Customers asking for discount, driving AOV down

ROOT CAUSES:
├─ Other sellers offer lower price
├─ Customer price shopping (normal)
├─ No perceived value difference
└─ Unlimited discounting culture

FIXES:
1. Never compete on price (compete on service)
2. Say: "Our price is fair for quality. Here's why..." (explain)
3. Counter with value, not discount:
   - "Free gift with purchase" (not price cut)
   - "Extra item included" (not 10% off)
   - "Faster delivery" (not cheaper)
4. Reserve discounts for:
   - Bulk orders (3+ packs)
   - Referrals (customer brings friend)
   - Loyalty (5th purchase)
5. Stance: "We keep prices fair and quality high"

═══════════════════════════════════════════════════════

❌ PROBLEM: Response time slipping >2 hours

ROOT CAUSES:
├─ Too many messages at peak times
├─ No batching system (reacting vs proactive)
├─ Chatbot not configured right
└─ Team member overloaded

FIXES:
1. Batch check messages: Every 30 min, respond all
2. Use chatbot for common questions (instant)
3. Set expectations: "We reply < 2h usually 7h-22h"
4. Hire second person if >100 messages/day
5. Audit: Where do messages really come from?
   (80% probably same 5 questions = auto-answer)

═══════════════════════════════════════════════════════

❌ PROBLEM: Customers not replying to follow-ups

ROOT CAUSES:
├─ Message not interesting (generic)
├─ Timing wrong (sent at 3am)
├─ Too many messages (spam fatigue)
├─ Wrong segment (offer not relevant)
└─ Customer genuinely lost interest (OK!)

FIXES:
1. Personalize: Use name "Fatima, t'aimes..."
2. Send at right time: 17h-20h mostly, not 8am
3. Reduce frequency: Max 1 message/day per customer
4. Segment first: Different message per customer type
5. Accept: Some people won't convert. That's OK!
6. Give up after 48h with respect: "No pressure!"
